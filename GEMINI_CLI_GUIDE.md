# 🤖 Руководство по использованию Gemini CLI (Fallback Mode)

Если у вас закончились токены в основном интерфейсе Antigravity, вы можете продолжать разработку прямо из терминала через `Gemini CLI`.

## 🚀 Быстрый старт

Запустите интерактивный чат, который понимает структуру вашего проекта:

```bash
gemini
```

### 🛠 Режимы редактирования (Самое важное!)

Чтобы CLI мог сам менять файлы (как агент), используйте флаги одобрения:

1.  **Безопасный агент**:
    ```bash
    gemini --approval-mode auto_edit
    ```
    *Позволяет ИИ редактировать файлы, но спрашивает разрешение на выполнение команд в терминале.*

2.  **Полный автомат (YOLO)**:
    ```bash
    gemini --yolo
    ```
    *ИИ будет вносить правки и выполнять команды без лишних вопросов.*

---

## 💡 Полезные сценарии

### 1. Редактирование конкретного файла
```bash
gemini -p "Добавь новую вкладку в ArchiveTab.tsx для отображения статистики" --yolo
```

### 2. Исправление ошибок (Линтинг)
```bash
gemini -p "Запусти npm run lint и исправь все найденные ошибки в папке src" --yolo
```

### 3. Написание тестов
```bash
gemini -p "Напиши unit-тесты для функции fetchTelegram в supabase папке" --approval-mode auto_edit
```

### 4. Резюме сессии
Если вы хотите продолжить предыдущий разговор в CLI:
```bash
gemini --resume latest
```

---

## ⚙️ Настройка и проверка

*   **API Ключ**: CLI использует `GEMINI_API_KEY` из вашего `.env.local` или системных переменных.
*   **Расширения**: У вас уже установлено расширение `conductor`, которое помогает управлять планами реализации.
*   **Контекст**: Если нужно добавить внешние папки в контекст:
    ```bash
    gemini --include-directories ../another-folder
    ```

---

## 🆘 Если что-то не работает
*   Проверьте версию: `gemini --version` (актуальная: `0.23.0`)
*   Список сессий: `gemini --list-sessions`
*   Помощь: `gemini --help`
